 Singleton is a creational design pattern that lets you ensure that a class has only one instance, while providing a global access point to this instance.

The Singleton pattern solves two problems \+:-\/

1 Ensure that a class has just a single instance.

2 Provide a global access point to that instance.\hypertarget{md__2home_2sagar_2design__pattern_2creational__design__patterns_2singleton_2Singeltion_autotoc_md1}{}\doxysection{\texorpdfstring{steps to create single design pattern}{steps to create single design pattern}}\label{md__2home_2sagar_2design__pattern_2creational__design__patterns_2singleton_2Singeltion_autotoc_md1}

\begin{DoxyItemize}
\item Make the default constructor private, to prevent other objects from using the new operator with the Singleton class.
\item Create a static creation method that acts as a constructor. Under the hood, this method calls the private constructor to create an object and saves it in a static field. All following calls to this method return the cached object. 
\end{DoxyItemize}
